# Объяснение RAG для диплома

## Что такое RAG?

**RAG (Retrieval-Augmented Generation)** — это архитектура ИИ-систем, которая комбинирует:
1. **Поиск информации** (Retrieval) — нахождение релевантных документов в базе знаний
2. **Генерацию ответа** (Generation) — создание ответа языковой моделью на основе найденного контекста

### Почему RAG лучше обычной LLM?

| Обычная LLM | RAG-система |
|-------------|-------------|
| Отвечает только на основе обучающих данных | Использует актуальные документы |
| Может "галлюцинировать" (выдумывать факты) | Ответы основаны на реальных источниках |
| Не знает специфичных данных организации | Работает с вашей базой знаний |
| Сложно обновить знания | Достаточно добавить документы в базу |
| Не указывает источники | Показывает, откуда взята информация |

## Компоненты системы

### 1. Document Ingestion (Загрузка документов)

**Цель:** Преобразовать PDF учебники в формат, пригодный для поиска.

**Процесс:**
```
PDF файл
   ↓
Извлечение текста (PyPDF2)
   ↓
Разбиение на чанки (500 символов с перекрытием 50)
   ↓
Векторизация (эмбеддинги через ChromaDB)
   ↓
Сохранение в векторную базу
```

**Код:**
```python
def chunk_text(text, chunk_size=500, chunk_overlap=50):
    """Разбивает текст на перекрывающиеся чанки."""
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        # Разбиваем по границе предложения
        chunk = text[start:end]
        chunks.append(chunk.strip())
        start = end - chunk_overlap
    return chunks
```

### 2. Vector Search (Векторный поиск)

**Цель:** Найти наиболее релевантные фрагменты учебников для заданного вопроса.

**Как работает:**
1. Вопрос пользователя преобразуется в вектор (эмбеддинг)
2. Вычисляются косинусные расстояния до всех документов в базе
3. Возвращаются топ-N ближайших документов

**Математика:**
```
cosine_similarity(A, B) = (A · B) / (||A|| × ||B||)
```

Где:
- `A` — вектор вопроса
- `B` — вектор документа
- Результат от -1 до 1 (чем ближе к 1, тем более релевантны)

**Код:**
```python
def query(self, query_text, n_results=5, subject_filter=None):
    """Поиск релевантных документов."""
    results = self.collection.query(
        query_texts=[query_text],
        n_results=n_results,
        where={"subject": subject_filter} if subject_filter else None,
        include=["documents", "metadatas", "distances"],
    )
    return results["documents"][0], results["metadatas"][0]
```

### 3. Context Augmentation (Обогащение контекста)

**Цель:** Сформировать промпт для LLM с найденным контекстом.

**Структура промпта:**
```
System: Ты — образовательный ассистент EduFarm...

User: 
Контекст из учебников:
Документ 1: [текст фрагмента из учебника]
Документ 2: [текст фрагмента из учебника]
...

Вопрос ученика: {query}

Дай развернутый ответ на русском языке, опираясь на контекст.
```

**Код:**
```python
def chat_with_context(self, query, context_documents):
    context_text = "\n\n".join([
        f"Документ {i+1}:\n{doc}" 
        for i, doc in enumerate(context_documents)
    ])
    
    user_message = (
        f"Контекст из учебников:\n{context_text}\n\n"
        f"Вопрос ученика: {query}\n\n"
        f"Дай развернутый ответ..."
    )
    
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": user_message},
    ]
    
    return await self.chat(messages)
```

### 4. LLM Generation (Генерация ответа)

**Цель:** Получить связный ответ от языковой модели.

**Используется:** локальная LLM через Ollama

**Запрос:**
```http
POST http://127.0.0.1:11434/api/chat
Content-Type: application/json

{
  "model": "qwen2.5-3b-instruct-q4km",
  "messages": [...],
  "stream": false,
  "options": {
    "temperature": 0.2,
    "num_ctx": 4096,
    "num_predict": 700
  }
}
```

## Преимущества выбранной архитектуры

### 1. Модульность
Каждый компонент независим:
- Можно заменить Ollama-модель на другую локальную LLM
- Можно заменить ChromaDB на другую векторную базу
- Можно изменить стратегию чанкования

### 2. Масштабируемость
- RAG-сервис вынесен в отдельный микросервис
- Горизонтальное масштабирование через Kubernetes
- Векторная база хранится отдельно

### 3. Обновляемость знаний
Для добавления новых знаний:
```bash
curl -X POST http://rag-service/textbooks/load
```
Не нужно переобучать модель!

### 4. Прозрачность
Ответы сопровождаются источниками:
```json
{
  "answer": "...",
  "sources": [
    {"textbook": "calculus", "page": 45},
    {"textbook": "algebra", "page": 112}
  ]
}
```

## Технические детали реализации

### Стек технологий

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| API Framework | FastAPI | Асинхронность, автодокументация, типизация |
| Vector DB | ChromaDB | Легковесная, локальная ONNX-векторизация, persistence |
| PDF Processing | PyPDF2 | Простота, надежность |
| HTTP Client | Httpx | Асинхронные запросы к Ollama |
| Logging | Loguru | Удобное структурированное логирование |

### Производительность

**Время обработки запроса:**
- Векторный поиск: ~50-100ms
- Запрос к Ollama: зависит от локальной модели и железа
- **Итого:** ~3-6 секунд

**Оптимизации:**
- Фильтрация по предмету уменьшает поиск
- Ограничение MAX_CONTEXT_DOCUMENTS=5

### Безопасность

1. **Аутентификация** — JWT от основной платформы
2. **CORS** — строгие правила для доменов
3. **Rate Limiting** — защита от злоупотреблений
4. **Валидация** — Pydantic модели для всех запросов

## Возможности расширения

### 1. Multi-modal RAG
Добавить поддержку изображений и формул из учебников.

### 2. Fine-tuning
Дообучить модель на конкретных учебных материалах.

### 3. Hybrid Search
Комбинировать векторный поиск с keyword search (BM25).

### 4. Reranking
Дополнительный этап ранжирования найденных документов.

### 5. Conversation History
Учет истории диалога для контекстных ответов.

## Выводы для диплома

RAG-архитектура выбрана потому что:

1. ✅ **Решает проблему галлюцинаций** LLM за счет привязки к источникам
2. ✅ **Позволяет использовать актуальные данные** без переобучения
3. ✅ **Обеспечивает прозрачность** через указание источников
4. ✅ **Масштабируема** — каждый компонент можно улучшать независимо
5. ✅ **Интегрируется** с существующей платформой EduFarm как микросервис

Эта архитектура представляет собой современный подход к созданию образовательных ИИ-ассистентов и может быть рекомендована для использования в аналогичных проектах.
